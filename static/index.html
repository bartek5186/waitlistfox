Waitlist API
with unsubscribe

Backend zapisuje użytkowników do waitlisty bez duplikatów e-mail, przechowuje metadane requestu i pozwala usunąć wpis po świadomym potwierdzeniu adresu oraz powodu wypisu.

Formularz obok wysyła żądanie do API dopiero po poprawnym uzupełnieniu wszystkich wymaganych pól.

GET /health Status aplikacji
POST /v1/waitlist/subscribe Dodaje adres do waitlisty
POST /v1/waitlist/unsubscribe Usuwa wpis po podaniu e-maila i powodu

Unsubscribe

Podaj adres e-mail i wybierz powód rezygnacji. Po poprawnym wysłaniu formularza wpis zostanie usunięty z waitlisty.

Formularz dołączy też bieżący czas, `user_agent` i `referrer`.
